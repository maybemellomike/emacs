* Define Straight.el and use-package

Hoping to keep this as portable as possible I was reading that using straight.el was a pretty clean way to do so.

straight.el~ is a declarative, reproducible package manager that clones
packages directly from their source repositories (Git) rather than downloading
pre-built tarballs from an archive. This means:

- Any package reachable by a Git URL can be installed, not just those on MELPA
- No ~package-refresh-contents~ on every startup — straight only hits the
  network when you explicitly ask it to
- Fully portable: on a new machine the first Emacs startup clones everything
  from scratch with no manual steps

~use-package~ is retained as the configuration frontend. The two integrate
via ~straight-use-package-by-default~, which makes every ~use-package~
declaration automatically route through ~straight.el~ for installation.
The ~:ensure t~ / ~:ensure nil~ flags from the old ~package.el~ setup are
no longer needed and have been removed throughout.

The bootstrap snippet below is the canonical straight.el self-installer. It
checks whether straight is already installed and, if not, clones it from
GitHub. This runs on every startup but is effectively instant once the file
exists.

#+BEGIN_SRC emacs-lisp
;; Bootstrap straight.el
(defvar bootstrap-version)
(let ((bootstrap-file
       (expand-file-name
        "straight/repos/straight.el/bootstrap.el"
        (or (bound-and-true-p straight-base-dir) user-emacs-directory)))
      (bootstrap-version 7))
  (unless (file-exists-p bootstrap-file)
    (with-current-buffer
        (url-retrieve-synchronously
         "https://raw.githubusercontent.com/radian-software/straight.el/develop/install.el"
         'silent 'inhibit-cookies)
      (goto-char (point-max))
      (eval-print-last-sexp)))
  (load bootstrap-file nil 'nomessage))

;; Tell straight.el to use use-package as its declaration frontend.
;; straight-use-package-by-default means every use-package form will
;; automatically install via straight without needing :straight t each time.
(straight-use-package 'use-package)
(setq straight-use-package-by-default t)
#+END_SRC

* Relative Line Numbers

I *do not for the life of me understand* what is going on here. I have taken now from so many configurations I just can't come to believe it's this complicated. I'm having some strange issues with this indenting text in strange spots because it appears the numbers are center aligned so when the go from one to two digits it pushes the text around as the cursor moves. Oh sorry, when /the point/ moves, around the file.


The keys to a stable gutter width are:
1. Set ~display-line-numbers-width~ globally as a default BEFORE the mode enables.
2. Set ~display-line-numbers-right-justify~ to ~t~ so numbers are right-aligned
   within the fixed-width gutter. Without this, numbers are left-aligned and the
   text body shifts sideways whenever the relative number gains or loses a digit
   as the cursor moves.
3. In org-mode, re-assert the width AFTER ~variable-pitch-mode~ fires, because
   proportional fonts cause Emacs to recalculate gutter metrics. We do this by
   appending to the hook (the ~t~ argument to ~add-hook~) so it runs last.

#+BEGIN_SRC emacs-lisp
(setq-default display-line-numbers-type 'relative)
(setq-default display-line-numbers-width 4)  ;; 4 = 3 digit columns + 1 padding so
                                              ;; two-digit numbers never reflow the gutter
(setq-default display-line-numbers-right-justify t) ;; right-align numbers so the text
                                                     ;; body stays anchored as digits change
(global-display-line-numbers-mode t)

(dolist (mode '(term-mode-hook
                shell-mode-hook
                eshell-mode-hook
               vterm-mode-hook
                treemacs-mode-hook
                minibuffer-setup-hook))
  (add-hook mode (lambda () (display-line-numbers-mode 0))))

;; Re-assert fixed gutter width in org-mode AFTER variable-pitch-mode fires.
;; The trailing `t` appends this hook so it runs after all others on the hook list.
(add-hook 'org-mode-hook
          (lambda ()
            (setq-local display-line-numbers-width 4)
            (setq-local display-line-numbers-width-start nil))
          t)

** Theme Toggle

Wish I could find a clean way to do this computer wide but hey, maybe Emacs will become my computer so this works. 

Toggles between the dark theme (~doom-monokai-machine~) and the light theme
(~ef-cyprus~). ~C-c T~ is used because ~C-c t~ is already bound to the eshell toggle.

~ef-cyprus~ is part of the ~ef-themes~ package by Protesilaos Stavrou. The toggle
tracks the current state via a simple variable so repeated presses always flip
between exactly the two themes and nothing else.

Because the ~org-block~ face background is not set by most themes — they leave it
to the user — we re-apply it explicitly after each toggle via
~my/apply-org-theme-faces~. This ensures code blocks always have an appropriate
tinted background on both dark and light themes rather than a jarring mismatch.

#+BEGIN_SRC emacs-lisp
(use-package ef-themes
  :custom
  (custom-safe-themes t))

(defvar my/dark-theme  'doom-monokai-machine
  "The dark theme to use in `my/toggle-theme'.")

(defvar my/light-theme 'ef-cyprus
  "The light theme to use in `my/toggle-theme'.")

(defvar my/current-theme 'dark
  "Tracks which theme is active: the symbol `dark' or `light'.")

(defun my/apply-org-theme-faces ()
  "Re-apply org code-block faces to match the currently active theme.
Called after every theme switch so blocks never inherit stale colours."
  (if (eq my/current-theme 'dark)
      ;; Dark theme — monokai-flavoured block colours
      (custom-set-faces
       '(org-block            ((t (:inherit fixed-pitch :background "#2d2d2d"))))
       '(org-block-begin-line ((t (:inherit fixed-pitch :foreground "#aaaaaa" :background "#2d2d2d"))))
       '(org-block-end-line   ((t (:inherit fixed-pitch :foreground "#aaaaaa" :background "#2d2d2d"))))
       '(org-code             ((t (:inherit fixed-pitch :foreground "#A6E22E")))))
    ;; Light theme — ef-cyprus uses warm off-whites; a pale grey block reads cleanly
    (custom-set-faces
     '(org-block            ((t (:inherit fixed-pitch :background "#eef0e8"))))
     '(org-block-begin-line ((t (:inherit fixed-pitch :foreground "#888c80" :background "#eef0e8"))))
     '(org-block-end-line   ((t (:inherit fixed-pitch :foreground "#888c80" :background "#eef0e8"))))
     '(org-code             ((t (:inherit fixed-pitch :foreground "#3a5f00")))))))

(defun my/toggle-theme ()
  "Toggle between `my/dark-theme' and `my/light-theme'."
  (interactive)
  (if (eq my/current-theme 'dark)
      (progn
        (mapc #'disable-theme custom-enabled-themes)
        (load-theme my/light-theme t)
        (setq my/current-theme 'light)
        (my/apply-org-theme-faces)
        (message "Switched to light theme: %s" my/light-theme))
    (progn
      (mapc #'disable-theme custom-enabled-themes)
      (load-theme my/dark-theme t)
      (setq my/current-theme 'dark)
      (my/apply-org-theme-faces)
      (message "Switched to dark theme: %s" my/dark-theme))))

;; Apply the correct faces for the initial dark theme on startup
(my/apply-org-theme-faces)

(global-set-key (kbd "C-c T") #'my/toggle-theme)
#+END_SRC

** Frame Transparency

This isn't working even though I'm on Wayland and transparency in everything else I have installed works without a hitch.

Sets the background to 90% opacity (10% transparent) using ~alpha-background~,
which is available in Emacs 29+. Unlike the older ~alpha~ parameter which fades
the entire frame including text, ~alpha-background~ leaves all text and UI
elements fully opaque — only the background colour bleeds through to whatever
is behind the Emacs window.

The value is a number from 0 (fully transparent) to 100 (fully opaque).
~default-frame-alist~ applies it to every frame including the first one,
whereas ~set-frame-parameter~ applies it to the already-created initial frame
that exists before ~default-frame-alist~ is processed.

Requires a compositor to be running. On NixOS with a desktop environment
(GNOME, KDE, etc.) this works out of the box. On a bare tiling window manager
you may need Picom or similar.

#+BEGIN_SRC emacs-lisp
(defvar my/frame-opacity 70
  "Background opacity for all Emacs frames, 0–100.")

;; Apply to the current frame immediately
(set-frame-parameter nil 'alpha-background my/frame-opacity)

;; Apply to every future frame (e.g. emacsclient frames)
(add-to-list 'default-frame-alist `(alpha-background . ,my/frame-opacity))
#+END_SRC

* Config Reload

~C-c r~ tangles ~config.org~ to ~config.el~ and hot-reloads it without restarting Emacs.

How it works:
- ~org-babel-tangle-file~ writes a fresh ~config.el~ from the org source blocks.
- ~load-file~ evaluates that file in the running Emacs session, applying every change
  immediately.
- The function is bound globally so it works from any buffer, not just when
  ~config.org~ is open.

One thing to be aware of: a hot-reload re-evaluates the whole file, which is fine
for settings and keybinds. Packages that were already loaded won't be unloaded —
~use-package~ will simply skip them as already-present, which is the correct
behaviour. If you ever add a brand new package and ~C-c r~ doesn't install it,
a single restart will sort it out.

#+BEGIN_SRC emacs-lisp
(defun my/reload-config ()
  "Tangle config.org to config.el and reload it without restarting Emacs."
  (interactive)
  (let ((org-file (expand-file-name "config.org" user-emacs-directory))
        (el-file  (expand-file-name "config.el"  user-emacs-directory)))
    ;; Delete the existing config.el before tangling so stale blocks from
    ;; previous versions of the config are never silently carried forward.
    (when (file-exists-p el-file)
      (delete-file el-file))
    (org-babel-tangle-file org-file el-file "emacs-lisp")
    (load-file el-file)
    (message "Config reloaded successfully.")))

(global-set-key (kbd "C-c r") #'my/reload-config)

(defun my/open-config ()
  "Open config.org in the current window."
  (interactive)
  (find-file (expand-file-name "config.org" user-emacs-directory)))

(global-set-key (kbd "C-c o c") #'my/open-config)
#+END_SRC

* Project Layouts

Each function below opens a purpose-built window layout for a specific project
or directory. They all follow the same pattern:

1. ~delete-other-windows~ for a clean slate
2. ~dired~ in the project directory in the top window
3. A visible eshell pane in the bottom third, ~cd~'d into the project directory
4. Focus returned to dired so you can navigate immediately

The ~C-c o~ prefix is used throughout: ~o~ for "open", followed by a letter
identifying the project.

A shared helper ~my/project-layout~ handles the common dired + eshell
scaffolding so each project function only needs to declare its directory,
buffer names, and any extra background processes.

#+BEGIN_SRC emacs-lisp
(defun my/project-layout (dir eshell-buf-name)
  "Open a standard project layout for DIR.
Creates a dired window on top and an eshell in a bottom split,
both rooted at DIR.  ESHELL-BUF-NAME is the buffer name to use
for the eshell so each project gets its own independent shell."
  (unless (file-directory-p dir)
    (user-error "Directory not found: %s" dir))
  (delete-other-windows)
  (dired dir)
  (let* ((total-height (frame-height))
         (shell-height (max 10 (/ total-height 3)))
         (eshell-win   (split-window-below (- total-height shell-height))))
    (select-window eshell-win)
    (if (get-buffer eshell-buf-name)
        (progn
          (switch-to-buffer eshell-buf-name)
          (with-current-buffer eshell-buf-name
            (eshell/cd dir)
            (eshell-reset)))
      ;; Create a new eshell with the desired buffer name
      (let ((eshell-buffer-name eshell-buf-name))
        (eshell 'new)))
    (with-current-buffer eshell-buf-name
      (eshell/cd dir)
      (eshell-reset)))
  (select-window (get-buffer-window (dired-noselect dir))))

(defun my/bg-process (name dir command &rest args)
  "Ensure a background process named NAME is running COMMAND in DIR.
If a process with NAME is already live, does nothing.
Output goes to a hidden buffer ' *NAME*' (leading space keeps it
out of the buffer list)."
  (unless (process-live-p (get-process name))
    (let ((default-directory dir))
      (apply #'start-process name (format " *%s*" name) command args)
      (set-process-query-on-exit-flag (get-process name) nil))))
#+END_SRC

* System Clipboard

Makes the ~"+"~ and ~"*"~ registers in Evil behave exactly like they do in Neovim —
~"+p~ pastes from the system clipboard and ~"+y~ yanks into it.

In Neovim this works because Neovim talks to the OS clipboard directly. Emacs has
its own internal kill-ring that is separate from the system clipboard by default.
The two settings below bridge that gap:

- ~select-enable-clipboard t~ makes every Emacs yank/kill also write to the OS
  clipboard, mirroring Neovim's ~clipboard=unnamed~.
- ~select-enable-primary t~ additionally syncs the X11 primary selection (the
  middle-click buffer on Linux). Remove this line if you are on macOS or Windows,
  or if you find it interferes with other workflows.
- ~evil-want-clipboard t~ (set in ~:init~ before Evil loads) tells Evil to route
  all yank and delete operations through the ~"+"~ register by default, exactly
  matching Neovim's ~clipboard+=unnamedplus~.

The ~xclip~ package is used as the backend on Linux because Emacs running inside a
terminal otherwise cannot reach the graphical clipboard. If you are running Emacs
as a GUI application (~emacs~ rather than ~emacs -nw~) it will still work correctly —
~xclip~ simply becomes a no-op in that context. On macOS the built-in ~pbcopy~/~pbpaste~
integration is used automatically and ~xclip~ is skipped.

#+BEGIN_SRC emacs-lisp
;; Bridge Emacs kill-ring to the OS clipboard
(setq select-enable-clipboard t)
(setq select-enable-primary t)   ;; Linux X11 primary selection — remove on macOS/Windows

;; Tell Evil to use the system clipboard register by default,
;; so plain `y` and `p` also go through "+", matching Neovim's
;; clipboard+=unnamedplus behaviour.
(setq evil-want-clipboard t)

;; xclip provides clipboard access when Emacs is running in a terminal on Linux.
;; Safe to keep on macOS — the package detects the platform and does nothing there.
(use-package xclip
  :config
  (xclip-mode 1))
#+END_SRC

** Terminal (eshell in a bottom split)

I am also always doing a lot of repetive CD'ing around so maybe there is a better just use other Emacs ways of doing things to avoid this. 

Opens an eshell in a horizontal split occupying the bottom third of the current
frame. If an eshell buffer is already open it reuses it rather than opening a
second one. Pressing ~C-c t~ again from any window closes the eshell split and
returns focus to the window you were in.

#+BEGIN_SRC emacs-lisp
(defun my/toggle-eshell-bottom ()
  "Toggle an eshell in a horizontal split at the bottom third of the frame."
  (interactive)
  (let* ((eshell-buf-name "*eshell*")
         (existing-win (get-buffer-window eshell-buf-name)))
    (if existing-win
        ;; If eshell window is already visible, close it and return focus
        (progn
          (delete-window existing-win)
          (message "Eshell closed."))
      ;; Otherwise open a new split sized to one third of the frame height
      (let* ((total-height (frame-height))
             (shell-height (max 10 (/ total-height 3)))
             (new-win (split-window-below (- total-height shell-height))))
        (select-window new-win)
        (if (get-buffer eshell-buf-name)
            (switch-to-buffer eshell-buf-name)
          (eshell))))))

(global-set-key (kbd "C-c t") #'my/toggle-eshell-bottom)
#+END_SRC

** Window Zoom (toggle fullscreen split)

~C-c f~ toggles the current window between fullscreen and the previous layout,
similar to tmux's ~prefix-z~. On the first press the current window configuration
is saved and ~delete-other-windows~ expands the focused window to fill the frame.
On the second press the saved configuration is restored exactly as it was.

~winner-mode~ is enabled to provide the configuration history that makes this
possible. It is a built-in minor mode that records window layout changes and is
also what allows ~C-c <left>~ / ~C-c <right>~ to step through layout history
manually if you ever want that.

#+BEGIN_SRC emacs-lisp
(winner-mode 1)

(defvar my/window-zoom--saved-config nil
  "Saved window configuration before zooming, or nil when not zoomed.")

(defun my/window-zoom-toggle ()
  "Toggle the current window between fullscreen and the previous layout.
Works like tmux prefix-z: first press expands, second press restores."
  (interactive)
  (if my/window-zoom--saved-config
      (progn
        (set-window-configuration my/window-zoom--saved-config)
        (setq my/window-zoom--saved-config nil)
        (message "Layout restored."))
    (setq my/window-zoom--saved-config (current-window-configuration))
    (delete-other-windows)
    (message "Window zoomed — C-c f to restore.")))

(global-set-key (kbd "C-c f") #'my/window-zoom-toggle)
#+END_SRC

* Padding

Emacs has no single "padding" setting, so three separate mechanisms are combined
to produce the effect:

- ~internal-border-width~ — a frame-level property that adds a gap of ~5px~ between
  the outer edge of the Emacs frame and the inner window area. This is the global
  outer padding and is set once on the default frame parameters.

- ~window-margins~ — per-window left and right gutters measured in character columns.
  Set to ~3~ columns on each side inside every window so text does not sit flush
  against the split border or the frame edge.

- ~header-line-format~ — normally used for a status bar at the top of a window.
  By setting it to a single space string it renders as a blank line, producing a
  top padding gap within each window. There is no built-in equivalent for bottom
  padding, but combined with the margins this breaks the flush appearance on all
  four sides.

All three are applied via ~window-configuration-change-hook~ so they reapply
automatically whenever a new split is created or a buffer is switched.

The ~header-line~ face is explicitly set to inherit the default background so it
renders as invisible padding rather than a visible grey bar. ~:box nil~ and
~:underline nil~ remove any border the theme may add to the face.

#+BEGIN_SRC emacs-lisp
;; Outer frame border: 5px gap between the OS window edge and Emacs' window area
(add-to-list 'default-frame-alist '(internal-border-width . 5))

;; Make the header-line face invisible — same background as the buffer,
;; no box, no underline, so it acts as silent top padding only.
(custom-set-faces
 '(header-line ((t (:inherit default
                    :background unspecified
                    :foreground unspecified
                    :box nil
                    :underline nil)))))

;; Apply per-window padding to every window whenever the layout changes
(defun my/set-window-padding ()
  "Add left/right margins and a blank header line to every window."
  (walk-windows
   (lambda (win)
     (unless (window-minibuffer-p win)
       ;; 3 columns of left and right margin inside every split
       (set-window-margins win 3 3)
       ;; Blank header line acts as top padding
       (with-current-buffer (window-buffer win)
         (setq-local header-line-format " "))))
   nil t))

(add-hook 'window-configuration-change-hook #'my/set-window-padding)

;; Run once immediately so padding is present before any layout changes occur
(my/set-window-padding)
#+END_SRC

* Spell Checking (jinx)

~jinx~ is a fast spell checker backed by the system ~enchant-2~ library, which
in turn delegates to whichever dictionary backend is installed (Hunspell,
Aspell, Nuspell etc.). It underlines misspelled words lazily as you type and
provides Vim-style next/previous navigation between errors.

The only OS dependency is ~enchant-2~ with at least one dictionary. On NixOS:

#+BEGIN_EXAMPLE
environment.systemPackages = [ pkgs.enchant pkgs.hunspellDicts.en-us ];
#+END_EXAMPLE

On Debian/Ubuntu: ~sudo apt install enchant-2 hunspell-en-us~.
On macOS (Homebrew): ~brew install enchant~ (dictionary bundled).

*How it works:*
- ~C-c s~ toggles ~jinx-mode~ on and off in the current buffer. When active,
  misspelled words are underlined.
- ~]s~ jumps to the next misspelled word (mirrors Vim's ~]s~).
- ~[s~ jumps to the previous misspelled word (mirrors Vim's ~[s~).
- ~C-c $~ opens a correction menu at point with suggested fixes (the default
  ~jinx~ correction bind, kept as-is since it does not conflict with anything).

The ~]s~ / ~[s~ bindings are added to ~evil-normal-state-map~ so they behave
exactly as they do in Vim — available globally whenever Evil is in normal
state, not just when ~jinx-mode~ is active, so the muscle memory is consistent.

#+BEGIN_SRC emacs-lisp
(use-package jinx
  :bind
  (("C-c s"   . jinx-mode)         ;; toggle spell checking in current buffer
   ("C-c $"   . jinx-correct)      ;; open correction menu at point
   :map evil-normal-state-map
   ("]s" . jinx-next)              ;; next misspelling  (Vim ]s)
   ("[s" . jinx-previous)))        ;; previous misspelling (Vim [s)
#+END_SRC
